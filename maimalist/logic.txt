## Minimalist 가계부 (SpendTrack) 설계도
# 1. 프로그램 개요
    목적: 사용자별 로그인 기능을 통해 개인화된 입출금 내역을 기록하고 통계를 확인하는 프로그램

    개발 환경: Python (모듈 분리 구조: spendtrack.py - 메인 로직, function.py - 기능 함수)  

# 2. 자료구조 (Data Structure)
    데이터의 연속성과 사용자 구분을 위해 다음과 같은 자료구조를 사용합니다.
    users (Dictionary): 사용자 이름을 키(Key)로, 현재 소지금을 값(Value)으로 저장합니다.
    예: {"김완중": 10000}
    
    history (List of Dictionaries): 모든 거래 내역을 딕셔너리 형태로 담는 리스트입니다.
    항목: 날짜, 유형(입금/출금), 항목명, 금액

    current_user (String/None): 현재 로그인한 사용자의 이름을 추적합니다.

# 3. 프로그램 상세 로직 (Logic Flow)
    단계 1: 시스템 초기화 및 로그인 메뉴
    프로그램 시작 시 로그인이 되지 않은 상태(current_user is None)에서는 초기 메뉴가 출력됩니다.

        1. 신규등록: 이름을 입력받아 users에 추가(중복 체크 포함)하고, 초기 소지금을 설정합니다.

        2. 로그인: users에 등록된 이름인지 확인 후, 일치하면 해당 사용자로 접속합니다.

        3. 종료: 루프를 빠져나가 프로그램을 완전히 종료합니다.

    단계 2: 로그인 후 메인 기능 (메인 루프)
    로그인 성공 후에는 해당 사용자의 이름과 잔액이 상단에 표시되며, 4가지 핵심 기능을 제공합니다.

        0. 입출금 내역 입력 (manage_money):
            입금 또는 출금을 선택합니다.
            날짜, 항목, 금액을 입력받습니다.
            로직: 입금 시 소지금 증가, 출금 시 소지금 감소(단, 잔액 부족 시 경고 후 중단).
            최종 데이터를 history 리스트에 저장합니다.

        1. 기간별 입출금 조회 (view_history):
            특정 날짜 패턴(예: "26.04")을 입력받습니다.
            **.startswith()**를 활용해 해당 기간에 맞는 내역만 필터링하여 출력합니다.
            해당 기간의 총 입금액, 총 출금액, 그리고 순수 합계를 계산하여 요약 리포트를 보여줍니다.

        2. 전체 거래 내역 확인 (view_all_history):
            사용자의 모든 기록을 날짜순으로 정렬(sorted)하여 리스트 형태로 보여줍니다.
            마지막에 현재 사용자의 실시간 총 잔액을 출력합니다.

        3. 종료: 메인 루프를 종료하고 프로그램을 마칩니다.

        4. 핵심 알고리즘 및 특징
            모듈화 (Modularization): 가독성을 위해 메인 실행 파일과 기능 정의 파일을 분리하여 관리합니다.
            예외 처리: 존재하지 않는 사용자 로그인 시도나 잔액을 초과하는 출금 시도 시 적절한 안내 문구를 출력합니다.

            유연한 검색: 문자열 비교 함수를 통해 사용자가 연도별 또는 월별로 자유롭게 검색할 수 있도록 구현했습니다.
